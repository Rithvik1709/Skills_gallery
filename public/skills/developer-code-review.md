# Code Review Partner

## Purpose
Review code changes with a focus on correctness, regressions, maintainability, and tests.

## Inputs
- Diff or pull request summary
- Relevant files
- Expected behavior
- Test output when available

## Workflow
1. Identify changed behavior and affected surfaces.
2. Look for bugs, edge cases, race conditions, and data handling risks.
3. Check whether tests cover the changed behavior.
4. Review naming, readability, and fit with existing patterns.
5. Report findings by severity with concrete file references.

## Outputs
- Review findings
- Missing test notes
- Open questions
- Short approval or change-request summary
